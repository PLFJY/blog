七、结尾

这次 Bug 最恶心的点在于：它不是一个单点错误，而是几个错误叠在一起：

能修复，又复发；

然后每一步都问清楚：

当前操作的是哪个用户的注册表？
当前 key 是锁着还是解锁？
现在请求的权限会不会被自己的 Deny 拦住？

这次修完以后我最大的感想是：

Windows 注册表 ACL 不是不能碰，但一定不能凭感觉碰。尤其是服务进程改用户 hive 的时候，每一步都必须把上下文和权限想明白。

不然它真的会变成一台非常昂贵的“权限地狱制造机”。
